# Supervisor Nudge Examples

These samples illustrate the professional, direct, and context-aware prompting style used by the Supervisor Nudge Protocol to re-engage inactive agents.

## Case 1: Silent Post-Implementation
**Context**: The agent has just finished writing a new function but hasn't run tests.
**Supervisor Nudge**: "Implementation complete. Please proceed to execute unit tests and verify the logic."

## Case 2: Stalled during Research
**Context**: The agent has searched for documentation but has been idle for 5 minutes.
**Supervisor Nudge**: "Research phase is reaching the threshold. Please summarize your findings and begin the implementation plan."

## Case 3: Error Loop
**Context**: The agent has encountered the same compilation error twice.
**Supervisor Nudge**: "The previous fix attempt resulted in a regression. Please re-analyze the symbol definitions in `registry.go` and provide an alternative solution."

## Best Practices
- **No Self-Reference**: Never say "As your supervisor..." or "I noticed you were idle."
- **Focus on Momentum**: Every nudge should point to the next logical action.
- **Brevity**: Keep nudges under 20 words where possible.
