# Self-Healing Go Tool Pipeline & Skill Ingestion System

This document outlines the architecture and workflows for maintaining compiler clean state in generated Go MCP tool wrappers, alongside the ingestion and Jaccard-based deduplication of agent skill libraries.

---

## 1. Compiler-Based Self-Healing Pipeline
The TormentNexus Go Command line wraps all generated MCP tools in `go/internal/tools/`. Because these files are generated by background LLM workers (Swarm), syntax and semantic errors can sometimes slip through.

To handle this autonomously, we run a compiler-driven self-healing reset loop:

### The Mechanism
1. **Compilation Check**: The [compiler_reset.py](file:///c:/Users/hyper/workspace/tormentnexus/scripts/compiler_reset.py) script is executed, which invokes the Go compiler on the main entrypoint:
   ```bash
   go build -buildvcs=false ./cmd/tormentnexus
   ```
2. **Error Parsing**: The script captures `stderr` and searches for compilation errors referencing generated files located in `internal/tools/`.
3. **Targeted Deletion**: Any generated `.go` file that causes compilation failures is immediately deleted from disk (excluding the critical core files listed in `PROTECTED_FILES`).
4. **Database Reset**: The database records in `data/assimilation_state.db` matching the failed files are updated:
   - `status` is set back to `'pending'`
   - `notes` is set to `'reset broken tool (compiler error)'`
   - `go_file` and `tools_exposed` are cleared.
5. **Re-Compilation Loop**: The script loops and repeats this process until the compiler returns a success status.
6. **Swarm Regeneration**: The background Swarm worker automatically claims the `'pending'` tasks and regenerates them.

## 2. Skill Ingestion & Jaccard Deduplication
The project utilizes a vast library of 2,950+ prompt/skill runbooks located in `~/.tormentnexus/skills/`. To make this library accessible to Go-native systems, we index them into a SQLite registry.

### The Mechanism
- **Path**: [ingest_all_user_skills.py](file:///c:/Users/hyper/workspace/tormentnexus/scripts/ingest_all_user_skills.py)
- **Database**: `.tormentnexus/skills.db`
- **Deduplication Threshold**: `90% Jaccard Similarity`

### Similarity Calculation
For each parsed skill, its word tokens (alphanumeric, length >= 2) are compared against all unique canonical skills already stored:
\[ Jaccard(A, B) = \frac{|A \cap B|}{|A \cup B|} \]
- **Duplicate (Similarity >= 90%)**: Stored with its `canonical_id` referencing the matched skill.
- **Canonical (Similarity < 90%)**: Stored as a new base skill with `canonical_id = NULL`.

This ensures the registry remains compact and optimized for context inclusion and progressive disclosure without duplicate bloat.
